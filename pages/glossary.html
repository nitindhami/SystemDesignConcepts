System Design Glossary

Short, interview-usable definitions for common architecture terms.

Schema

Structured definition of stored data, constraints, and relationships.

Replication

Keeping copies of data across nodes for availability and read scale.

Sharding

Splitting data across partitions to distribute load and storage growth.

Consistency

How quickly all readers see the latest write across the system.

Latency

Time taken for one request to receive a response.

Throughput

Amount of work handled per second (e.g., requests per second).

Idempotency

Safe retry behavior where repeated requests produce the same result.

Backpressure

Technique to slow producers when consumers cannot keep up.

Tip: Use glossary terms in your interview narration to sound precise and structured.
